## 一、概述

数据质量平台已完成身份验证与安全加固，包含真实 JWT、API 全面鉴权、登录限流、密码强度校验等。本文档说明环境变量及部署安全建议。

## 二、环境变量

| 变量 | 说明 | 默认值 | 生产建议 |
|------|------|--------|----------|
| `SECRET_KEY` | JWT 签名密钥 | 随机生成 | **必须** 配置不少于 32 字节随机字符串 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Token 有效期（分钟） | 1440（24 小时） | 按安全策略调整 |

### 生成 SECRET_KEY

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

## 三、部署安全建议

1. **SECRET_KEY**：生产环境必须通过环境变量或 `.env` 配置，禁止使用默认随机值
2. **HTTPS**：生产环境使用 HTTPS，启用 HSTS（需反向代理配置）
3. **CORS**：`CORS_ORIGINS` 仅配置可信前端域名，避免 `*`
4. **数据库**：生产使用 PostgreSQL/MySQL，SQLite 仅适用于开发
5. **密码策略**：新建用户密码至少 8 位，须含字母和数字

## 四、认证与鉴权

- **白名单**（无需登录）：`/auth/login`、`/auth/login/form`、`/auth/ping`、`/`
- **需登录**：`/auth/me`、`/datasources/*`、`/rule-sets/*`、`/tasks/*`、`/reports/*`、`/profiling/*`、`/users/*`
- **需管理员**：用户管理的增删改操作

## 五、登录限流

- 每 IP 每分钟最多 5 次登录尝试
- 超限返回 HTTP 429

## 六、默认账号

- 运行 `seed_data.py` 后默认账号：**admin / admin123**
- 生产环境首次部署后请立即修改默认密码
