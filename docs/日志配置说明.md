# 日志配置说明

**编辑时间**: 2025-03-14  
**代码编写人**: Lambert tang  

---

## 一、概述

数据质量平台采用集中式、可配置的日志管理。支持按环境变量配置日志级别、输出目标（控制台/文件）、格式与轮转策略。

## 二、环境变量

在项目根目录或 `backend/` 下的 `.env` 文件中配置：

| 变量 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `LOG_LEVEL` | 日志级别 | `INFO` | `DEBUG`、`INFO`、`WARNING`、`ERROR` |
| `LOG_FILE` | 日志文件路径，不设则仅输出到控制台 | `None` | `logs/app.log` |
| `LOG_FORMAT` | 日志格式 | `%(asctime)s [%(levelname)s] %(name)s: %(message)s` | - |
| `LOG_ROTATION` | 文件轮转策略 | `midnight`（按日） | `midnight` 或 `10MB`（按大小） |

### 示例配置

```bash
# 开发环境：仅控制台，DEBUG 级别
LOG_LEVEL=DEBUG

# 生产环境：文件 + 控制台，ERROR 级别
LOG_LEVEL=ERROR
LOG_FILE=logs/app.log
LOG_ROTATION=midnight

# 按大小轮转（10MB）
LOG_FILE=logs/app.log
LOG_ROTATION=10MB
```

## 三、日志级别建议

| 环境 | LOG_LEVEL | 说明 |
|------|-----------|------|
| 开发 | `DEBUG` | 便于调试，输出详细规则执行等 |
| 测试 | `INFO` | 正常业务日志，请求/任务/数据源操作 |
| 生产 | `INFO` 或 `WARNING` | 减少噪音，保留关键操作与异常 |
| 生产（高负载） | `ERROR` | 仅保留错误，减少 I/O |

## 四、输出说明

- **控制台**：始终输出到 stderr，格式统一
- **文件**：当 `LOG_FILE` 设置时，同时写入文件
  - `midnight`：按日轮转，保留 7 天
  - `10MB` 等：按大小轮转，保留 5 个备份
- **uvicorn.access**：单独限制为 WARNING，减少访问日志噪音

## 五、CLI 与脚本

- **main.py**（CLI 工具）：使用 `logging.basicConfig`，仅控制台、INFO 级别
- **seed_data.py**、**init_db.py**：作为脚本运行时，同样使用 basicConfig
- **backend 服务**：通过 `app.core.logging_config.setup_logging_from_settings` 初始化，遵循 `.env` 配置

## 六、主要日志点

| 模块 | 级别 | 场景 |
|------|------|------|
| 请求中间件 | INFO/WARNING/ERROR | 方法、路径、状态码、耗时、客户端 IP |
| auth | INFO | 登录成功/失败（不记录密码） |
| tasks | INFO | 任务开始、完成、数据源数 |
| datasources | INFO | 创建/删除数据源，测试连接结果 |
| data_loader | WARNING/ERROR | 加载失败、文件为空、解析错误 |

## 七、文件与目录

- 配置模块：`backend/app/core/logging_config.py`
- 请求日志中间件：`backend/app/middleware/request_logging.py`
- 日志目录：`logs/`（已加入 `.gitignore`）
